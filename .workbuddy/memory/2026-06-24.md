## astock-peg 服务启动问题（反复断开）

**问题根因**：WorkBuddy 沙箱环境会杀掉所有通过 Bash 工具启动的后台进程，导致 `nohup`、`disown`、`screen`、`launchctl bootstrap` 都无法持久化服务。

**解决方案**：创建一键启动脚本 `astock-peg/start.sh`，用户需在 **自己的终端（iTerm/Terminal）** 中手动运行：
```bash
bash ~/Claude/大A投研看板/astock-peg/start.sh
```

**已配置内容**：
- `.env` 已配置 DeepSeek API Key (`OPENAI_API_KEY`, `BASE_URL=https://api.deepseek.com`, `MODEL=deepseek-chat`)
- Python 使用系统 Anaconda `/opt/anaconda3/bin/python3`
- Node.js 使用 managed 版本 22.22.2

**管理命令**：
```bash
# 启动
bash ~/Claude/大A投研看板/astock-peg/start.sh
# 检查
lsof -nP -i :3000
# 停止
kill $(lsof -ti :3000)
# 日志
tail -f /tmp/astock-peg.log
```

- `refreshing` 状态控制 800ms 内按钮禁用 + 旋转动画

- **修复 CAGR 计算错误（严重）**：`calcValuation` 和 `nYearGrowth` 中 3年/5年 CAGR 使用了**单季度**数据而非 TTM 数据
  - `calcValuation` 的 g3：`cur.deducted_profit` → `cur.ttm?.deducted_profit`（沪电股份修正：85.36% → 51.06%）
  - `calcValuation` 的 gCurrent：也改为 TTM 口径
  - `nYearGrowth` 的 revenue/profit CAGR：全部改为 `cur.ttm?.xxx ?? cur.xxx` 取 TTM
  - 原因：单季度数据季节性波动大（如 Q1 往往和 Q4 差距很大），TTM 滚动12个月更能反映真实年化增长
- tsc 0 错误

- **新增 astock-peg 估值标签（section 十一开头）**：对有研报的股票，使用彼得·林奇 PEG 法估算合理股价
  - PEG = 前瞻PE / (归母净利3年TTM CAGR × 100)
  - 合理价(PEG=1) = 当前股价 / PEG
  - EPS来源：优先同花顺一致预期；无则从研报eps_this_year取均值（≥2家有效研报才启用）
  - 6 个 MetricCard：一致预期EPS、前瞻PE、PEG(林奇)、合理股价、净利润CAGR(3年)、PE消化年限
  - PEG 评级色标：<0.5极度低估(绿) / 0.5~1低估(绿) / 1~1.5合理(灰) / >1.5偏贵(红)
  - EPShandle：研报接口返回字符串，前端用 Number() 转换
- tsc 0 错误

- **PEG 估值分析改为所有股票均显示（section 十一独立 section）**：
  - 原 astockPeg 标签从研报 section 中拆出，成为独立 section 十一「PEG 估值分析」
  - 原研报 section 改为 section 十二
  - **EPS 三级回退链**：①同花顺一致预期(最优) → ②研报 eps_this_year 均值(≥2家) → **③TTM 实际 EPS = TTM归母净利/总股本（兜底）**
    - 总股本 ≈ 市值/股价，EPS = 净利润(亿元)×10^8 / 总股本(亿股)
  - **CAGR 双口径回退**：归母净利 CAGR > 0 时优先使用；≤0 或 null 时回退到扣非净利 CAGR
  - 当 CAGR 无效时，PEG/前瞻PE/合理价/消化年限显示 "N/A"，EPS 和 CAGR 数值仍可展示
  - 新增 `epsSource` 字段标记数据来源（一致预期 / 研报均值 / TTM 实际），在 EPS 标签 sub 中显示
