## 个股看板 TTM 数据修复

**问题**：用户反馈 TTM 营收/利润数据不对（如 601888 显示 1386 亿，但实际年营收 564 亿）。

**根因**：新浪 `CompanyFinanceService.getFinanceReport2022` 返回的是**累计**报告期数据
（Q1累计=167.46、H1累计=281.51、Q3累计=398.62、年报累计=536.94），但后端
`_build_fundamentals_from_statements` 直接把它当**单季**用，导致 TTM = 169+537+399+282 = 1386亿
（错误地把累计值重复加和）。

**修复**：在 builder 中识别报告期月份（03/06/09/12），用"本期YTD - 上期YTD"得到单季值：
- 03-31: 单季 = YTD
- 06-30: 单季 = H1累计 - Q1累计
- 09-30: 单季 = Q3累计 - H1累计
- 12-31: 单季 = 年报累计 - Q3累计

**验证**（中国中免 601888）：
- 2026Q1 单季 = 169.06亿 ✅（与新闻一致）
- 2025Q4 单季 = 138.31亿 ✅（与新闻一致）
- TTM 营收 = 538.54亿 ✅
- TTM YoY = -1.03%（同口径，2026Q1 vs 2025Q1=167.46，新闻说同比+0.96%）

**注意**：TTM YoY 改成"对齐 TTM"是错误的——TTM 538 vs 去年 TTM（2025Q1+2024Q4+2024Q3+2024Q2
= 167+135+118+125 = 545）= -1.1%，恰好和 -1.03% 接近。所以原 TTM YoY 逻辑也是对的。

## 关键教训

1. **WorkBuddy sandbox 会杀所有非 launchctl 启动的后台进程** — nohup & disown & setsid 都不行，只有 launchctl + LaunchAgents 才行
2. **企业网下 curl 必须 `env -u HTTP_PROXY -u HTTPS_PROXY -u http_proxy -u https_proxy`** — shell 默认有 `HTTP_PROXY=http://127.0.0.1:xxxxx`
3. **launchctl 启动 vite 必须设 `EnvironmentVariables.PATH`** — launchd 不继承 shell PATH
4. **plist 必须放 `~/Library/LaunchAgents/`** — /tmp 下报 "Load failed: 5"
