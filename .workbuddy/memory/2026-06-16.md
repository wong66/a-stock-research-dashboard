# 2026-06-16 17:21 vibe-trading 个股看板 修复"显示不全"

## 用户反馈
"上方和左侧都有缩" —— 截图显示：
- 左侧 aside 里的"我的自选股"标题被 main 边缘截断
- 分组标签"A 股 (2)" 只显示 "(2)"
- list 项里股票名"中国中免""兆易创新"被截掉，只剩 [A股] [601888] [58.4] [-1.15%]
- 但**右侧财务图表正常显示**（说明数据通路是好的）

## 根因
1. **StockBoard 外层用了 `-m-4 md:-m-8`**（试图把内边距顶到 Layout main 边缘）—— 但 Layout main 没有 padding，负 margin 反而让子内容溢出
2. **左侧 aside 宽度 `w-80 lg:w-96`** 太宽（320-384px），加上 padding 后股票名 truncate 空间不够
3. **右侧 sticky header 紧贴 main 左边**，看着像被 main 截断

## 修复
- StockBoard 外层 `h-[calc(100vh-3.5rem)] -m-4 md:-m-8` → `h-full min-w-0`
- aside `w-80 lg:w-96` → `w-[19rem]`，加 `min-w-0` 防止 flex 子项不收缩
- StockRow：
  - `gap-3 px-4 py-3` → `gap-2 px-3 py-2.5`（紧凑）
  - price `min-w-[5rem]` → `min-w-[3.5rem]`
  - percent `min-w-[4.5rem]` → `min-w-[3.75rem]`
  - 移除 `ChevronRight` 装饰（占空间）
  - 名字加 `font-medium` 和 `title=` tooltip，hover 时显示完整名
- 移除 `ChevronRight` 的 import

## 教训
- Layout.tsx 的 main 是 `flex-1 overflow-auto`，**子页面不要再用 h-[calc(100vh-...)]**（会与外层 scroll 容器冲突）
- 子页面的 -m-x 负 margin **只在父级有对应 padding 时**才生效；本项目 main 没有 padding，所以 -m-4 是反效果
- **关键陷阱**：右侧 detail 面板里 ECharts 卡片（grid-cols-2 md:grid-cols-4）会让 `<main>` 内容区**比视口宽**，Layout 的 main 是 `overflow-auto`（双轴），整个页面就会**横向滚动**，导致左侧 aside 被推出屏幕左侧
- 解决：StockBoard 根加 `w-full overflow-hidden`，右侧 detail `<main>` 加 `min-w-0 overflow-x-hidden` —— 把溢出关在面板内

# 2026-06-16 17:30 vibe-trading 个股看板 再次修复（右侧溢出导致左侧被推走）

## 用户反馈
"未修改好" —— 截图显示：
- 左侧 aside 仍被横向推出屏幕左侧，"我的自选股"变成"自选股"，"A 股 (1)"变成"(1)"
- 右侧财务图表正常显示
- 行内价格、涨跌幅齐全，但**股票名仍然被截断**

## 根因
Layout 的 `<main className="flex-1 overflow-auto">` 是**双轴可滚动**。右侧 detail 面板里的 `grid-cols-2 md:grid-cols-4` 卡片组在窄屏下不收缩，整个右侧 main 实际渲染宽度 > 视口宽度。
→ Layout main 出现横向滚动条
→ 整个 StockBoard（包括左侧 aside）一起被横向滚动
→ 左侧 aside 起点 = -(溢出宽度)，开头的 "我的"/"A 股" 文字被裁掉

之前的修改（aside `min-w-0`）只让 aside 内容收缩，但**没让根容器阻止溢出**。

## 修复
1. StockBoard 根：`<div className="flex h-full min-w-0">` → `<div className="flex h-full w-full overflow-hidden">`
2. aside：`w-[19rem] ... min-w-0` → `w-72 ... shrink-0`（固定 288px，加 shrink-0 防止被压）
3. 右侧 main：`<main className="flex-1 overflow-y-auto">` → `<main className="flex-1 min-w-0 overflow-y-auto overflow-x-hidden">`
   - `min-w-0` 让 flex 子项能小于内容宽度
   - `overflow-x-hidden` 阻止横向滚动

## 经验
- 在 flex 容器里要做"左侧固定宽度 + 右侧自适应"布局时，**中间一定要加 `min-w-0`**，否则右侧会撑爆整个 flex 容器
- 横向 overflow 默认是 `visible`，任何子元素超出都会被父级**撑大**——即使父级有 `overflow:hidden` 也只在父级的 box 内裁切，子元素不会自动收缩
- Layout 用 `overflow-auto` 而非 `overflow-hidden`，意味着子页面**必须自己管理溢出**

## 后端新增 `_build_ttm_yoy(periods)` (api_server.py)
- 在 `_build_fundamentals_from_statements` 之后调用一次
